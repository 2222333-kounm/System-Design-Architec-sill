要给「System-Design-Architec-sill」这个项目加上「无限画布」功能，思路非常契合这个项目「将抽象视觉灵感转化为结构化的代码约束」的目标。它能让设计规范的展示不再受限于传统网页的边界，转而用一种更直观、富有探索感的方式呈现。

建议采用轻量级的无限画布库来快速实现核心功能，同时将项目原有的设计系统（Design System）无缝地映射到画布世界中。

---
### 为何推荐使用 `@ue-too/board` 等轻量库？

主要推荐使用 `@ue-too/board`、`@niuee/board` 这类专注「画布能力」的库，原因如下：

*   **功能纯净**：它们不包含特定的 UI 组件或绘图工具，只专注于提供最核心的无限画布能力：**平移（Pan）**、**缩放（Zoom）** 和 **旋转（Rotate）**。这与项目本身的功能定位相符。
*   **核心优势**：
    *   **简单易用**：API 设计直观，初始化极快，能轻松地将普通的 HTML `<canvas>` 或 DOM 元素转变成具备无限画布特性的交互空间。
    *   **性能优异**：它们不强制全量重绘，通常会通过 Camera 抽象来处理坐标转换，并支持视口剔除和按需渲染，确保流畅的交互体验。
    *   **框架无关**：不仅支持原生 JavaScript，也能很好适配 React、Vue 等主流框架，灵活性极高。
*   **其他可选方案**：
    *   **重量级 SDK**：`tldraw SDK` 功能强大，能快速构建复杂白板，但它体积较大、定制成本高，对「快速为静态页面添加交互」的场景可能显得过重。
    *   **渲染引擎**：`LeaferJS` 基于 GPU 加速，性能极致，但学习曲线更陡峭。
    *   **Canvas 库**：`Fabric.js` 常用于对象编辑，但原生并不直接支持「无限画布」的视口概念，需要手动实现很多逻辑。

### 集成方案

#### 第一步：搭建基础架构
在项目目录中新建 `infinite-canvas-demo.html`，搭建一个基础的 HTML 框架，并预留画布容器。

#### 第二步：引入轻量级库（`ue-too/board` 为例）
通过 CDN 引入是演示和开发最快捷的方式：
```html
<!-- 官方推荐的基础样式 -->
<link rel="stylesheet" href="https://unpkg.com/@ue-too/board@latest/dist/style.css">
<script type="importmap">
    {
        "imports": {
            "@ue-too/board": "https://esm.sh/@ue-too/board"
        }
    }
</script>
<script type="module">
    // 在此处编写初始化代码
</script>
```
#### 第三步：初始化画布并注入设计内容
编写 JavaScript 代码来完成核心的集成工作：
1.  **创建“相机”**：实例化 `Board` 类，它将接管指定容器，创建无限画布、处理所有手势交互和坐标转换。
2.  **加载内容**：将原先在 `site/index.html` 中定义好的、基于 Design Tokens 的内容块（如产品卡片、模块等），作为独立组件映射到画布上。甚至可以编写脚本，根据 `DESIGN.md` 文件自动生成这些内容节点。
3.  **提供视觉参考**：添加一个可无限延伸的网格背景，这能提供清晰的空间感，体现「无限」的概念。

#### 第四步：打造极致体验（高级优化）
1.  **确保超高帧率交互**：实现流畅的 120fps 平移和缩放，配合视差效果，创造沉浸式的「空间感」。
2.  **集成标准布局辅助工具**：为画布添加可动态调整的**标尺（Rulers）** 和**动态网格**，这能很好地体现一个「设计系统架构师」项目对精确性和可度量性的追求。
3.  **提供无障碍导航**：添加一个「重置视图」按钮或小地图（Minimap），方便用户快速将视角重置到画布中心或定位到特定内容区域。
4.  **实现多页/场景管理**：模仿 `dgm.js` 的多页面支持能力，在单个无限画布上通过不同区域来展示不同的设计概念、用户流程或产品变体，非常适合系统设计的演示。

这个方案将项目从一个静态的网页范例，升级为一个可探索的动态演示空间，使其「结构化设计约束」的理念在无限画布上得到更具沉浸感的体现。
